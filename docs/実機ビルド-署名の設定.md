# 実機ビルドが失敗する場合（署名エラーの対処）

次のエラーが出ている場合の対処です。

- **No Accounts: Add a new account in Accounts settings.**
- **No profiles for 'com.mahjong.mahjongMobile' were found**

---

## 手順1: Xcode に Apple ID を追加する

1. **Xcode** を起動する
2. メニュー **Xcode** → **Settings…**（または **Preferences…**）を開く
3. 左側の **Accounts** タブをクリック
4. 左下の **+** をクリック → **Apple ID** を選ぶ
5. **Apple ID** と **パスワード** でサインイン（無料の Apple ID で可）
6. 一覧に自分の Apple ID が表示されれば OK（**Team** に「Personal Team」などが出る）

---

## 手順2: プロジェクトの署名を設定する

1. ターミナルで次を実行して Xcode でプロジェクトを開く:
   ```bash
   open /Users/goto/WorkCase/mahjong-mobile/ios/Runner.xcworkspace
   ```
2. 左のプロジェクトナビゲーターで **Runner**（青いアイコンのプロジェクト）をクリック
3. 中央で **TARGETS** の **Runner** を選択
4. 上端の **Signing & Capabilities** タブを開く
5. **Automatically manage signing** にチェックを入れる
6. **Team** で、手順1で追加した Apple ID の **Personal Team**（または自分のチーム）を選択
7. **Bundle Identifier** が `com.mahjong.mahjongMobile` のままならそのままでよい  
   （「この ID は使用できません」と出る場合は、例: `com.yourname.mahjongMobile` のように変更する）
8. **ビルドの実行**（次のいずれか）
   - **キーボード**: **⌘ + B**（Command + B）を押す
   - **メニュー**: 画面上部 **Product** → **Build** をクリック
   - 画面下部のビルド結果に「Build Succeeded」と出れば成功

9. 実機にインストールして実行する場合
   - 画面上部の実行先（「Runner」の隣）で **GOTO's iPhone** を選択
   - **⌘ + R**（または **Product** → **Run**）で実機にインストール＆起動

---

## 手順3: ターミナル（Flutter）で再度実行する

Xcode でビルドが通ったら、ターミナルで:

```bash
cd /Users/goto/WorkCase/mahjong-mobile
flutter run -d "GOTO's iPhone" --dart-define=SERVER_URL=http://192.168.0.107:3001
```

初回は iPhone に「信頼されていない」や「開発者を信頼しますか？」と出ることがあります。  
→ 下記 **「iPhone で開発者を信頼する」** の手順を行ってください。

---

## iPhone で「信頼されていない」と出る場合

アプリを開こうとすると「信頼されていない開発元」と表示され、起動できない場合の手順です。

1. iPhone の **設定** アプリを開く
2. **一般** をタップ
3. **VPNとデバイス管理**（または **プロファイルとデバイス管理**）をタップ
4. **デバイス管理** の下に、開発者（自分の Apple ID の名前や「Developer App」など）が表示されるので、それをタップ
5. **「〇〇を信頼」**（〇〇は開発者名）をタップ
6. 確認ダイアログで **信頼** をタップ
7. ホーム画面に戻り、**Mahjong Mobile** アプリを再度タップして起動する

これでアプリが起動するようになります。同じ開発者（同じ Apple ID）でビルドしたアプリは、一度信頼すれば以降はこの操作は不要です。
